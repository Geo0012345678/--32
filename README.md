# Ечкин — Заметки с облачной синхронизацией

Итоговый проект по курсу WinForms (.NET 10). Настольное приложение для ведения
заметок с блокнотами, тегами, поиском, авторизацией по ролям и имитацией
облачной синхронизации.

---

## 1. Как открыть и запустить

1. Открой файл **`NotesCloudApp.sln`** в Visual Studio (нужен **.NET 10**).
2. При первом открытии Visual Studio сама восстановит NuGet-пакеты
   (Newtonsoft.Json, PdfSharp, NUnit, Moq). Если нет — правый клик по решению →
   «Восстановить пакеты NuGet».
3. Нажми **F5** (или зелёную кнопку «NotesCloudApp»). Запустится окно входа.
4. При первом запуске рядом с программой автоматически создаются папки `Data`
   и `Cloud` с демонстрационными данными.

## 3. Главное окно

Три вкладки:
- **📝 Заметки** — список заметок с поиском и фильтрами.
- **📚 Блокноты** — управление блокнотами (скрыта для роли «Читатель» и гостя).
- **☁ Синхронизация** — журнал операций обмена с облаком.

Внизу — строка состояния: текущее действие, состояние облака, имя и роль пользователя.

### Подсветка заметок по статусу
- **Белая** — синхронизирована с облаком.
- **Жёлтая** — есть несохранённые в облаке изменения («Только локально» / «Ожидает отправки»).
- **Красная** — конфликт версий.

## 4. Основные действия

### Заметки
- **Добавить** — кнопка «＋ Добавить» или клавиша **Insert**.
- **Изменить** — двойной щелчок или **F2**.
- **Удалить** — **Delete** (только администратор).
- Поле «Поиск» ищет по заголовку, тексту и тегам.
- Список «Блокнот» и флажок «Только избранное» фильтруют список.

### Блокноты
- Создание/изменение — «＋ Блокнот» / Insert / F2.
- Блокнот нельзя удалить, пока в нём есть заметки.

### Синхронизация с облаком
1. Меню **Данные → Синхронизация с облаком…** (или вкладка ☁ → «Синхронизировать»).
2. Выбери стратегию разрешения конфликтов: «самая свежая» / «локальная» / «облачная».
3. Нажми «Синхронизировать».
- Для демонстрации конфликта: в окне синхронизации выбери заметку и нажми
  «Изменить» (имитация правки с другого устройства), затем измени её локально и
  снова синхронизируй — увидишь обнаружение и разрешение конфликта.

### Отчёты
Меню **Отчёты → Экспорт отчёта** → выбери формат **CSV**, **HTML** или **PDF** → укажи путь.

### Отмена действий
- **Ctrl+Z** — отменить, **Ctrl+Y** — повторить (до 50 шагов).

### Сброс данных
Меню **Данные → Сбросить демонстрационные данные…** (только администратор) —
восстанавливает свежий демо-набор (20 заметок, 6 блокнотов).

## 6. Архитектура

Приложение построено по схеме **MVC** (Model–View–Controller) с отдельным слоем сервисов.

```
NotesCloudApp/
├── Models/        — данные: Note, Notebook, SyncOperation, User, Enums
├── Services/      — DataService (JSON), CloudSyncService, ExportService,
│                    UndoManager, ConflictResolutionStrategy, Logger, SecurityHelper
├── Controllers/   — NoteController, NotebookController, SyncController, AuthController
└── Forms/         — LoginForm, MainForm, NoteForm, NotebookForm, SyncForm,
                     ReportForm, ClassDiagramForm, AboutForm
```

**Паттерны проектирования:**
- **Observer** — события `DataChanged` / `StatusChanged` в контроллерах.
- **Factory Method** — `ReportFactory` создаёт отчёты CSV/HTML/PDF.
- **Memento** — `UndoManager` хранит снимки для Undo/Redo.
- **Strategy** — фильтрация заметок и стратегии разрешения конфликтов.

Диаграмму классов можно посмотреть прямо в программе: **Справка → Диаграмма классов**,
а также в файле `Документация/Architecture.md`.

## 9. Хранение данных

- `Data/notes.json`, `notebooks.json`, `sync.json`, `users.json` — локальные данные.
- `Cloud/cloud_notes.json` — имитация облачного хранилища.
- `log.txt` — журнал работы приложения.

Пароли хранятся только в виде SHA-256-хэша, в открытом виде нигде не сохраняются.
